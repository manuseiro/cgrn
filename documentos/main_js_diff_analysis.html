Acúmulo de listeners em searchCARByCode — delegação aplicada ✓ corrigido
🔁
Padrão event delegation + state.lastCarSearch
A solução adotada é correta e elegante. Os listeners dos botões "Importar" e "Ver" saíram de dentro de searchCARByCode (onde se acumulavam a cada chamada) e migraram para um único listener delegado no container #carSearchResult, registrado uma só vez em bindEvents(). O dado do CAR é passado via state.lastCarSearch.
// ANTES — listener adicionado a cada busca (acumulava): document.getElementById('btnImportarCAR') ?.addEventListener('click', () => { /* usa closure de `car` */ }); // DEPOIS — registrado uma vez em bindEvents(): document.getElementById('carSearchResult') ?.addEventListener('click', e => { const btnImport = e.target.closest('#btnImportarCAR'); if (btnImport) { const carData = state.lastCarSearch; // dado compartilhado via state // ... } }); // Em searchCARByCode(): state.lastCarSearch = car; // armazena resultado
⚠
state.lastCarSearch não declarado em state.js
A propriedade é adicionada dinamicamente ao objeto state, o que funciona porque ele não é Object.freeze()-ado. Porém, declarar explicitamente em state.js garante documentação, consistência com os demais campos e facilita ferramentas de análise estática.
// state.js — adicionar na declaração: export const state = { // ... carLayer: null, lastCarSearch: null, // ← adicionar esta linha };
