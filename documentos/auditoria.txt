Melhoria proposta para changelog.html e docs.html
Os arquivos já estão razoavelmente bem estruturados e usam Bootstrap + style.css, mas têm problemas de layout:

Container muito estreito e sem boa responsividade.
Falta de altura total com scroll suave.
Pouca consistência visual com o index.html (navbar, cards, sombras, espaçamentos, tema dark/light).
No changelog, a timeline pode crescer muito sem scroll controlado.
No docs, o sidebar sticky funciona, mas o conteúdo principal é curto e sem "cuerpo" visual.

1. Melhoria Geral (aplicar nos dois arquivos)
Adicione/substitua no <head> e <body> para maior fidelidade ao projeto:

[HTML]

<!-- No <head>, após os links existentes -->
<style>
    html, body {
        height: 100%;
        margin: 0;
        overflow: hidden; /* Controlado pelo container principal */
    }
    
    .main-container {
        height: calc(100vh - 80px); /* Ajusta para navbar */
        overflow-y: auto;
        scroll-behavior: smooth;
        padding-bottom: 2rem;
    }
    
    .card {
        border: none;
        border-radius: var(--cgrn-radius, 12px);
        box-shadow: var(--cgrn-shadow, 0 4px 20px rgba(0,0,0,0.1));
    }
    
    [data-bs-theme="dark"] .card {
        background-color: #1f2937;
    }
    
    .timeline {
        padding-left: 2.5rem;
    }
    
    /* Scrollbar moderna */
    .main-container::-webkit-scrollbar {
        width: 8px;
    }
    .main-container::-webkit-scrollbar-thumb {
        background: var(--bs-primary);
        border-radius: 4px;
    }
</style>
[/HTML]
